# Database Schema Documentation

## Project Overview 
this database is designed for a **role-based registration and login system** with **multi-level verification**.

## Roles in the system 
1. **Admin** - System owner, final authority
2. **higher authority** - Intermediate verifier 
3. **Shopkeeper** - Registers beneficiaries and shops 
4. **Beneficiary (end user)** - final user of the system 

Registration and access are controlled throught **strict verification flows**

## Database design Principles
- Role-based access control
- multilevel verification
